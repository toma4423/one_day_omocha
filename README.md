# 今日のおもちゃ

Streamlitを使用した、様々なおもちゃ（ミニゲーム・ツール）を集めたアプリケーションです。

## 実行方法
```bash
pip install -r requirements.txt
streamlit run app.py
```

## プロジェクト構成
本プロジェクトはStreamlitのマルチページアプリ構成を採用しています。

- **`app.py`**: アプリケーションのメインエントリポイントです。ホーム画面を表示します。
- **`pages/`**: 各おもちゃのページファイルが格納されています。
    - ファイル名は `番号_アイコン_名前.py` の形式（例: `1_🎲_サイコロ.py`）になっています。
    - Streamlitは自動的にこのディレクトリ内のファイルを読み込み、サイドバーにナビゲーションを作成します。

## 開発ガイドライン（AI向け）

### 新しいおもちゃの追加方法
新しいおもちゃ（機能）を追加する場合は、以下の手順に従ってください。既存の `app.py` を編集して分岐を追加するのではなく、**新しいページファイルを作成**してください。

1. **ファイル作成**: `pages/` ディレクトリ内に新しいPythonファイルを作成します。
    - ファイル名の命名規則: `{連番}_{アイコン}_{機能名}.py`
    - 例: 現在 `5_...` まである場合、次は `6_🚀_新機能.py` とします。

2. **基本テンプレート**:
   新しいファイルには以下の構成を含めてください。
   ```python
   import streamlit as st
   
   # ページ設定（タイトルとアイコン）
   st.set_page_config(page_title="機能名", page_icon="🚀")
   
   # セッション状態の初期化（必要な場合）
   # 他のページと競合しないユニークなキーを使用するか、
   # ページ固有の変数を適切に管理してください。
   if 'unique_key' not in st.session_state:
       st.session_state.unique_key = 0
       
   st.title("🚀 機能名")
   
   # ここにロジックを実装
   ```

3. **セッション状態の管理**:
   - 各ページは独立して再実行されますが、`st.session_state` はアプリケーション全体で共有されます。
   - 他のページで使用されているキー名と被らないように注意してください（例: `dice_total` や `board_data` など、汎用的な名前を使う場合は既存のコードを確認してください）。

4. **ライブラリの追加**:
   - 新しいライブラリが必要な場合は `requirements.txt` に追加してください。

### リファクタリング・修正について
- 各機能は独立したファイル (`pages/*.py`) に分離されています。特定の機能のバグ修正や改善を行う場合は、対象のファイルのみを編集してください。
- 共通機能やグローバルな設定を変更する場合のみ `app.py` を触る必要がありますが、基本的には各ページファイル内で完結させることを推奨します。
